# Отчет о проекте: RAG-система для учебной подготовки

## Тема

Разработка и исследование системы учебной подготовки на основе Retrieval-Augmented Generation. Проект реализован в виде fullstack-приложения «Зачетка», которое помогает работать с пользовательскими учебными материалами: загружать документы, искать ответы по источникам, генерировать пересказы, вопросы, карточки, мнемоники и конкурсные тренировочные режимы.

## Актуальность

Обычный чат с языковой моделью может отвечать уверенно, но без опоры на конкретные документы пользователя. Для учебной и конкурсной подготовки это создает риск неточных объяснений, выдуманных фактов и ответов, которые сложно проверить. RAG-подход снижает этот риск: модель получает не весь произвольный контекст, а найденные фрагменты из загруженных источников и формирует ответ с учетом этих фрагментов.

Проект ориентирован на задачи, где важны проверяемость и связь с исходным материалом:

- подготовка по конспектам, лекциям, лабораторным, методичкам и презентациям;
- повторение материала через карточки, вопросы и мнемоники;
- тренировка по нормативным и конкурсным документам;
- быстрый поиск ответа внутри набора файлов.

## Цель проекта

Цель проекта - разработать прототип RAG-системы для учебной подготовки и описать исследовательскую основу для оценки качества ответов по сравнению с обычным чатом без retrieval.

## Объект и предмет исследования

Объект исследования - процесс подготовки пользователя по собственным учебным и нормативным материалам.

Предмет исследования - применение RAG-подхода для генерации ответов и учебных активностей, основанных на найденных фрагментах документов.

## Гипотеза

Если языковая модель получает релевантные фрагменты из загруженных материалов и ограничивается ими при генерации ответа, то ответы становятся более проверяемыми, а риск галлюцинаций снижается по сравнению с режимом обычного чата.

## Реализованная система

«Зачетка» состоит из backend-приложения на FastAPI и frontend-интерфейса на React. Пользователь загружает один или несколько файлов, после чего система извлекает текст, делит его на смысловые фрагменты, строит эмбеддинги и сохраняет их в локальное векторное хранилище ChromaDB.

Основные возможности:

- мультизагрузка документов;
- обработка PDF, DOCX, TXT, MD, PPTX, XLSX, PNG и JPG;
- OCR для изображений при наличии Tesseract;
- RAG-чат по коллекции документов;
- генерация краткого учебного пересказа;
- генерация экзаменационных вопросов;
- генерация карточек для повторения;
- генерация мнемоник;
- конкурсные режимы `blitz`, `two_to_one`, `union_biathlon`;
- отображение источников, на которых основан ответ.

## Архитектура решения

```text
Файлы пользователя
  -> загрузка через frontend
  -> FastAPI backend
  -> извлечение текста
  -> очистка и разбиение на chunks
  -> локальные embeddings
  -> ChromaDB
  -> retrieval по запросу пользователя
  -> генерация ответа или учебной активности
  -> отображение результата и источников
```

Технологический стек:

- backend: FastAPI, Pydantic, sentence-transformers, ChromaDB;
- frontend: Vite, React, TypeScript, CSS;
- LLM: Caila через OpenAI-compatible API;
- embeddings: локальная модель `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2`;
- хранение пользовательских файлов и производных данных: локальные директории `data/uploads`, `data/processed`, `data/vector_store`.

## Методика исследования

В исследовательской части предполагается сравнение трех режимов:

- baseline chat - модель отвечает без доступа к документам;
- RAG chat - модель отвечает на основе найденных фрагментов;
- RAG generation - система создает учебные активности только из retrieved chunks.

Качество предлагается оценивать по следующим метрикам:

- accuracy - корректность ответа;
- completeness - полнота ответа относительно эталона;
- groundedness - подтверждаемость ответа источниками;
- hallucination rate - доля утверждений без основания в документах;
- citation precision - точность выбранных источников;
- citation recall - полнота найденных источников;
- time to answer - время до получения полезного ответа;
- perceived usefulness - субъективная полезность для пользователя.

Для экспертной оценки используется шкала 0-2 по критериям корректности, полноты, опоры на источники, отсутствия лишних утверждений и ясности объяснения.

## Результаты разработки

В рамках проекта подготовлен рабочий прототип, который закрывает полный пользовательский сценарий:

1. Загрузка нескольких материалов.
2. Автоматическая обработка файлов.
3. Создание коллекции документов.
4. Семантический поиск по коллекции.
5. Ответы в чате с источниками.
6. Генерация учебных и конкурсных активностей.
7. Отображение результатов в web-интерфейсе.

Исследовательская база вынесена в `RAG.ipynb` и `docs/research_notes.md`. Архитектура и API дополнительно описаны в `docs/architecture.md` и `docs/api.md`.

## Ограничения

Текущая версия является локальным исследовательским прототипом. Ограничения:

- качество retrieval зависит от качества извлечения текста из документов;
- OCR для изображений требует установленного Tesseract;
- длинные нормативные документы требуют аккуратного чанкинга;
- экспертная оценка качества ответов может быть субъективной;
- локальное хранилище подходит для прототипа и экспериментов, но не является промышленной системой хранения.

## Вывод

Проект показывает, как RAG можно использовать для учебной подготовки по пользовательским материалам. В отличие от обычного чата, система связывает ответы с загруженными источниками и позволяет строить учебные активности на основе конкретных документов. Такой подход делает подготовку более проверяемой и создает основу для дальнейшего исследования качества ответов, снижения галлюцинаций и повышения полезности генеративных систем в образовании.

